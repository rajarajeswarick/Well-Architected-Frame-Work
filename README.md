<img width="1839" height="997" alt="image" src="https://github.com/user-attachments/assets/1f00bf06-158f-4c2c-ba4d-049210304467" />
Check with the product owners on the priorities and then answer the questions
Reliability pillars is the most important pillar in System Architecture, Scalability, high availability and Disaster Recovery falls under this.
Question and Answers have additional info, click on the 'info' to know more about the question or answer.
Involve product owners and application team to answer the questions. 
<img width="1883" height="983" alt="image" src="https://github.com/user-attachments/assets/a76e66ff-2104-4dc6-9564-04be83b35d18" />
If a pillar is not applicable to  an application then we can skip that pillar.
Once this is done, a report is generated showing the gaps in the pillars, based on that Design can be fixed.
On the main page we can see Well- Architected lenses for some services and applications, example serverless askes extra questions since serverless workload is different from the workload running your machine.
